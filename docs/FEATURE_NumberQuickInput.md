# Feature Spec: 數字快打模式 (Number Quick Input Mode)

> **適用專案**：ThomasHsieh/vChewing-macOS（fork 自 vChewing/vChewing-macOS）  
> **參考來源**：自然輸入法 V12 數字快打功能、實機截圖  
> **目標模組**：`vChewing_Typewriter`、`vChewing_MainAssembly4Darwin`  
> **優先實作範圍**：Phase 1（數字轉換）→ Phase 2（計算機）→ Phase 3（日期/時間）
> **實作狀態**：✅ 全部 Phase 已完成（2026.04.05）

---

## 1. 功能概述

數字快打是一個**內嵌於正常組字流程中的輸入子模式**。使用者在組字區中（無論是否已有中文內容）連按兩下漂鍵 `` ` ``，組字區會追加「數字快打」標示並彈出提示窗，接著輸入數字/運算式/日期/時間，即可選出多種格式轉換結果，送出後與組字區前方的中文內容合併輸出。

**核心設計原則**：
- 組字區前方的已輸入中英文內容**不會被清空**，數字快打附加在後方
- 雙擊漂鍵後組字區顯示「數字快打」文字作為模式提示
- 使用者輸入數字/算式後，候選窗即時更新
- 最終送出時，前方中文 + 數字快打結果一併提交給應用程式

## 2. 觸發機制

### 2.1 漂鍵定義

| 項目 | 說明 |
|------|------|
| 按鍵 | `` ` ``（backtick，鍵盤左上角，keyCode 50） |
| 是否需搭配修飾鍵 | 否，直接按（按 Shift 是 `~`，不觸發） |
| 支援的鍵盤排列 | 大千傳統、微軟、零壹（`.ofStandard`）；ETen26（`.ofETen26`） |
| 不支援的排列 | 其他所有排列（切換至其他排列時此功能自動停用） |

> **鍵盤排列說明**：大千、微軟、零壹在 Tekkon 內部使用 `.ofStandard` 解析器；ETen26 使用 `.ofETen26` 解析器。`` ` `` 鍵在這些排列下均未映射任何注音符號，可安全作為漂鍵。

### 2.2 觸發流程

```
使用者第一下按 ` （漂鍵）
        ↓
InputHandler 偵測到 keyCode 50，且當前鍵盤排列為 .ofStandard 或 .ofETen26，且無修飾鍵
→ 記錄第一下時間戳記，暫不動作，等待第二下
        ↓
使用者在 300ms 內再次按 ` （第二下漂鍵）
        ↓
確認雙擊成立，進入數字快打模式
        ↓
組字區在原有內容後方追加「數字快打」文字（模式標示）
        ↓
彈出模式提示浮窗（見 3.1）
        ↓
等待使用者輸入數字/算式/日期/時間

【逾時處理】若第二下未在 300ms 內到達：
→ 將第一下的 ` 字元直接送出給應用程式，結束等待
```

### 2.3 觸發時的組字區狀態

| 觸發前組字區 | 觸發後組字區顯示 |
|-------------|-----------------|
| 空白 | `數字快打` |
| `測試` | `測試數字快打` |
| `已更新到最新版` | `已更新到最新版數字快打` |

「數字快打」四個字是**模式標示**，非輸入內容，以不同顏色或底線區分（參考現有組字區樣式實作）。

## 3. UI 規格

### 3.1 模式提示浮窗（雙擊漂鍵後立即顯示）

雙擊漂鍵後，彈出一個**小型說明浮窗**（非選字候選窗），內容如下：

```
數字快打
・數字（例如：``123）
・日期（例如：``2020.5.6）
・時間（例如：``13:00）
・算數（例如：``20*300）
```

> 此浮窗為純說明性 UI，不可選取。使用者開始輸入第一個有效字元後，浮窗自動關閉並改為顯示候選窗。

### 3.2 輸入後的候選窗

使用者輸入第一個有效字元後，改為顯示**田所選字窗**（TDKCandidateController），列出各種格式候選。

**候選項目格式**（以 `123456` 為例，依截圖順序）：

```
123,456
壹貳參仟肆佰伍拾陸元整
十二萬三千四百五十六
12萬3,456元
一二三四五六
壹貳參肆伍陸
```

### 3.3 組字區即時顯示

| 使用者輸入 | 組字區顯示 |
|-----------|-----------|
| 漂鍵後尚未輸入 | `[前方中文]數字快打` |
| 輸入 `123456` | `[前方中文]123456` |
| 輸入 `20*300` | `[前方中文]20*300 = 6000`（計算機模式即時預覽） |
| 輸入 `2020.5.6` | `[前方中文]2020.5.6` |

## 4. 四大子功能規格

### 4.1 數字轉換

**觸發條件**：輸入純數字序列（`0–9`）。

**候選清單**（以 `123456` 為例，依截圖順序）：

| 順序 | 格式 | 輸出 |
|------|------|------|
| 1 | 千分位 | `123,456` |
| 2 | 中文大寫財務（含元整） | `壹貳參仟肆佰伍拾陸元整` |
| 3 | 中文小寫完整讀法 | `十二萬三千四百五十六` |
| 4 | 混合格式 | `12萬3,456元` |
| 5 | 逐位中文小寫 | `一二三四五六` |
| 6 | 逐位中文大寫 | `壹貳參肆伍陸` |

**單一數字（1 位）的候選清單**（以 `5` 為例）：

| 順序 | 格式 | 輸出 |
|------|------|------|
| 1 | 阿拉伯數字 | `5` |
| 2 | 中文小寫 | `五` |
| 3 | 中文大寫 | `伍` |
| 4 | 全形 | `５` |
| 5 | 序數（圓圈） | `⑤` |
| 6 | 羅馬數字（1–10） | `Ⅴ` |

**中文數字對照表**：

```
小寫：零一二三四五六七八九
大寫：零壹貳參肆伍陸柒捌玖
單位（小寫）：十百千萬億兆
單位（大寫）：拾佰仟萬億兆
```

**轉換規則**：
- `10` → `十`（不加「一十」）；`110` → `一百一十`
- 混合格式：滿萬換中文單位，不足萬保留數字+千分位（例：`12萬3,456元`）
- 財務大寫結尾固定加「元整」
- 最大支援到兆（10^12）

---

### 4.2 算盤（計算機）

**觸發條件**：輸入含運算子的表達式（如 `20*300`）。

**支援運算子**：

| 鍵入 | 運算 |
|------|------|
| `+` | 加法 |
| `-` | 減法 |
| `*` | 乘法 |
| `/` | 除法 |
| `(` `)` | 括號 |
| `.` | 小數點 |

**即時計算**：每次按鍵後若語法完整，組字區右側顯示 `= <結果>` 預覽。

**候選清單**（以 `20*300=6000` 為例）：

| 順序 | 輸出 |
|------|------|
| 1 | `6000` |
| 2 | `6,000` |
| 3 | `20*300=6000` |
| 4 | `六千` |
| 5 | `陸仟` |

**錯誤處理**：
- 除以零 → 候選窗顯示「無法計算（除以零）」，不可送出
- 語法不完整 → 候選窗暫不顯示，等待完整輸入

---

### 4.3 日期格式轉換

**觸發條件**：輸入符合日期樣式的字串（含 `.` 或 `/` 分隔，例如 `2020.5.6`）。

**支援輸入格式**：

| 輸入格式 | 範例 |
|----------|------|
| `年.月.日` | `2020.5.6`、`113.5.6` |
| `年/月/日` | `2020/5/6`、`113/5/6` |
| `年-月-日` | `2020-5-6` |

**民國 vs 西元判斷**：年份 ≤ 150 → 民國年；年份 > 150 → 西元年。

**候選清單**（以 `2020.5.6` 為例）：

| 順序 | 輸出 |
|------|------|
| 1 | `2020年5月6日` |
| 2 | `民國109年5月6日` |
| 3 | `109年5月6日` |
| 4 | `2020/05/06` |
| 5 | `2020-05-06` |
| 6 | `二○二○年五月六日` |

---

### 4.4 時間格式轉換

**觸發條件**：輸入含 `:` 的時間字串（如 `13:00`）。

**支援輸入格式**：`HH:MM`（24 小時制）、`HH:MM:SS`

**候選清單**（以 `13:00` 為例）：

| 順序 | 輸出 |
|------|------|
| 1 | `13:00` |
| 2 | `下午1點00分` |
| 3 | `下午1:00` |
| 4 | `13時00分` |
| 5 | `十三點整` |

**AM/PM 判斷**：
- 0–4 時 → 凌晨
- 5–11 時 → 上午
- 12 時 → 中午
- 13–23 時 → 下午

## 5. 離開機制

| 操作 | 行為 |
|------|------|
| `Enter` | 送出選中候選結果（含前方中文），離開模式 |
| `Space` | 確認第一個候選（含前方中文），離開模式 |
| `↑` / `↓` / `←` / `→` | 在候選窗中移動選取（委派給 `handleCandidate`，更新 `highlightedCandidateIndex`）|
| `Esc` | 取消數字快打緩衝區，回到純中文組字狀態（前方中文保留） |
| `Backspace` | 刪除數字緩衝區最後一字元；若緩衝區已空，退出數字快打模式（移除標示） |
| 輸入注音鍵（非數字快打有效字元） | 先送出目前候選（若有），再處理注音輸入 |

## 7. 架構與實作位置

### 7.1 新增 IMEState

```swift
case ofNumberInput(
    precedingText: String,   // 數字快打前方的已組中文（可為空字串）
    numberBuffer: String,    // 使用者輸入的數字/算式/日期/時間（可為空字串）
    candidates: [String],    // 格式化後的候選清單
    displayHint: String?     // 即時計算結果預覽（如 "= 6000"），nil 表示無
)
```

### 7.2 新增檔案

| 檔案路徑 | 用途 |
|----------|------|
| `Packages/vChewing_Typewriter/Sources/Typewriter/NumberQuickInput/NumberQuickInputHandler.swift` | 主邏輯：漂鍵偵測、緩衝管理、格式分類、候選產生 |
| `Packages/vChewing_Typewriter/Sources/Typewriter/NumberQuickInput/NumberConverter.swift` | 數字格式轉換（千分位、中文大小寫、逐位、混合、財務） |
| `Packages/vChewing_Typewriter/Sources/Typewriter/NumberQuickInput/DateTimeConverter.swift` | 日期/時間格式偵測與轉換 |
| `Packages/vChewing_Typewriter/Sources/Typewriter/NumberQuickInput/ExpressionEvaluator.swift` | 四則運算求值（純 Swift，不依賴外部 framework） |

### 7.3 整合點（InputHandler）

在 `InputHandler.handleInput()` 最前端加入攔截邏輯：

```swift
// 1. 若已在數字快打模式，所有按鍵交由 NumberQuickInputHandler 處理
if numberQuickInputHandler.isActive {
    return numberQuickInputHandler.handle(event: event)
}

// 2. 偵測漂鍵雙擊觸發（僅 .ofDachen 排列且功能已啟用）
if event.keyCode == 50              // ` 鍵
    && !event.hasAnyModifierFlags   // 無修飾鍵
    && currentParser == .ofDachen
    && PrefMgr.shared.numberQuickInputEnabled {

    let now = Date().timeIntervalSince1970
    if let firstTap = lastBacktickTimestamp,
       now - firstTap <= 0.3 {
        // 雙擊成立
        lastBacktickTimestamp = nil
        numberQuickInputHandler.activate(precedingText: currentComposerText)
        return .absorbed
    } else {
        // 第一下，記錄時間戳記，暫不動作
        lastBacktickTimestamp = now
        return .absorbed  // 吸收第一下，等待第二下；逾時後在計時器回呼中送出 `
    }
}
```

> **注意**：需要在 `InputHandler` 中新增 `lastBacktickTimestamp: TimeInterval?` 屬性，並設置一個 300ms 的計時器；若計時器觸發時仍未收到第二下，則將 `` ` `` 字元送出給應用程式並清除時間戳記。

### 7.4 整合點（SessionCtl）

`ofNumberInput` 狀態下的 UI 渲染：
- **緩衝區為空時**（剛按漂鍵）：組字區顯示 `precedingText + "數字快打"`，顯示模式提示浮窗
- **有內容時**：組字區顯示 `precedingText + numberBuffer + (displayHint ?? "")`，顯示田所候選窗

### 7.5 平台閘控

所有新檔案以 `#if canImport(Darwin)` 閘控，確保 Linux 編譯不受影響。

## 9. 測試案例

### 數字

| 輸入 | 預期候選（部分） |
|------|----------------|
| `5` | `5`、`五`、`伍`、`⑤`、`Ⅴ` |
| `10` | `10`、`十`、`拾` |
| `1000` | `1,000`、`一千`、`壹仟` |

| 輸入 | 預期結果候選 |

| 輸入 | 預期候選（部分） |
|------|----------------|
| `2020.5.6` | `2020年5月6日`、`民國109年5月6日`、`2020/05/06` |

| 輸入 | 預期候選（部分） |
|------|----------------|

| （空） | `14:30`，選下午格式 | `下午2點30分` |

## 10. 不在本次範圍內（Out of Scope）

- 分數輸入（`1/2` → `½`）
- 貨幣符號自動前綴（NT$、USD）
- 算式歷史記憶
- 數字快打與智慧中英文切換的組合使用情境
